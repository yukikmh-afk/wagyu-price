# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
- リモートリポジトリ:Githubのクラウド上に保存される
- ローカルリポジトリ:各作業者の端末上に保存される
- リモートリポジトリに保存されているソースコードなどをローカルリポジトリに落としたり、
ローカルリポジトリ内の編集したソースコードをリモートリポジトリにアップロード(commit)することで共有することができる


## プッシュとマージの違いは何でしょうか？
- push:add,commitにより保存した差分をリモートリポジトリに反映させる。
- merge:ブランチ間の差分を取り込む。現在のbranchに取り込む元となるbranchを指定して差分を取り込み反映させる


## コミットとプッシュの違い
- commit:リモートリポジトリで更新された差分履歴を保存
- push:差分をリモートリポジトリに反映させる
- addで差分のあるファイルを指定 commitで差分を保存 pushでリモートに反映

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
- 自身がどのような差分(変更)を行ったのかが分かりやすく記述する


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
- 個人で勝手にマージする為変更点の共有が難しい
- プルリクエストを送ることで管理者が変更点を確認してからマージするため、変更点などに問題があったときに気が付いてマージを止められる可能性がある


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
- 二つのファイル内の同じ場所を変更して競合が発生してしまっているので解決策として以下２通り
1. どちらかの変更点のみを取り込む
2. 両方の変更点を取り込む
